# html-reader-guide.md — Single-HTML reader spec

In v0.2 the dual HTML (`papers/{id}.html` + `readers/{id}-reader.html`)
is collapsed into a **single `papers/{id}.html`** that serves as both
authoritative notes and pedagogical reader. sync.sh renders it from
`notes/{id}.md` + `papers.json`. This file specifies what that single
HTML must contain + the reusable render patterns.

## Target layout (top to bottom)

| Region | Source | Render |
|---|---|---|
| Top bar | fixed site nav | canonical `<nav>` with Papers / Knowledge / Graph / Tools / GitHub |
| Paper header | `papers.json` title / authors / date / category / tags | h1 + meta row |
| Hero figure | notes §3 first diagram OR `images/{id}/fig1.png` | base64-inlined PNG OR inline Mermaid |
| TL;DR card | notes §1 | prominent boxed card, ≤ 300 chars |
| Q1 / Q2 / Q3 cards | notes §2 | 3-column grid on desktop, stacked on mobile |
| Section nav | TOC from notes headings | sticky left rail with IntersectionObserver highlight |
| Body | notes §3–§9 | markdown render, with each §3 diagram rendered inline (Mermaid live / drawio lazy-load / PNG base64) |
| Images | `images/{id}/*.png` referenced by `### Figure N:` in notes | base64 inlined adjacent to the text that references it |
| Math | `$...$` and `$$...$$` in notes | KaTeX auto-render |
| Footer | "Back to index" + "Next / Previous paper" | generated by sync.sh |

Orphan images (no `### Figure N:` heading) are dropped. Local paths
(`/home/...`, `~/.cursor/...`) are sanitized out of the output.

## Render patterns (reusable, copy-paste into sync.sh / generate_html.py)

### Mermaid inline

```html
<script type="module">
import mermaid from "https://cdn.jsdelivr.net/npm/mermaid@11/dist/mermaid.esm.min.mjs";
mermaid.initialize({
  startOnLoad: true, theme: 'base', securityLevel: 'loose',
  themeVariables: {
    fontFamily: "'IBM Plex Sans', 'Noto Sans SC', sans-serif", fontSize: '14px',
    primaryColor: '#eff6ff', primaryTextColor: '#1e293b',
    primaryBorderColor: '#2563eb', lineColor: '#475569',
    secondaryColor: '#fef3c7', tertiaryColor: '#f0fdf4', background: '#ffffff'
  },
  flowchart: { htmlLabels: true, curve: 'basis' }
});
mermaid.run({ querySelector: '.mermaid' });
</script>
```

Source ` ```mermaid ` fenced blocks in notes get wrapped as
`<pre class="mermaid">…</pre>` by sync.sh.

### drawio embed — lazy-load pattern

Never use inline `data-mxgraph="{huge escaped JSON}"` — it silently fails
above ~10 KB drawios. Always use the lazy-load pattern:

```html
<div id="drawio-xml-{id}-{tag}" class="drawio-xml-source"
     style="display:none !important;" aria-hidden="true">{HTML-escaped drawio XML}</div>

<div class="mxgraph mxg-lazy"
     data-xml-source="drawio-xml-{id}-{tag}" data-page="0"
     style="width:100%;aspect-ratio:1500/900;border:1px solid #ddd;
            border-radius:8px;background:#fff;overflow:hidden;"></div>
```

Bootstrap JS at end of `<body>`:

```html
<script defer>
document.addEventListener('DOMContentLoaded', function () {
  document.querySelectorAll('.mxgraph.mxg-lazy').forEach(function (el) {
    var src = document.getElementById(el.dataset.xmlSource);
    if (!src) return;
    el.setAttribute('data-mxgraph', JSON.stringify({
      highlight: '#0000ff', nav: true, resize: false,
      toolbar: 'zoom layers tags lightbox pages', 'toolbar-position': 'top',
      fit: 1, 'auto-fit': 1, border: 5,
      'page-visible': false, lightbox: false, edit: '_blank',
      xml: src.textContent.trim(),
      page: parseInt(el.dataset.page || '0', 10)
    }));
    el.classList.remove('mxg-lazy');
  });
});
</script>
<script defer src="https://viewer.diagrams.net/js/viewer-static.min.js"></script>
```

### KaTeX math

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/contrib/auto-render.min.js"
  onload="renderMathInElement(document.body, {
    delimiters: [
      {left: '$$', right: '$$', display: true},
      {left: '$',  right: '$',  display: false}
    ],
    throwOnError: false
  });"></script>
```

All formulas in notes MUST use `$...$` / `$$...$$` (no Unicode math
glyphs outside fenced code blocks).

## Design spec

- **Fonts** (Google Fonts CDN only): Lora (English body), IBM Plex Sans
  (UI), Noto Serif SC (Chinese body), Noto Sans SC (Chinese UI),
  JetBrains Mono (code)
- **Colors**: page `#FAF8F3`, nav `#1B2A4A`, accent `#2563eb`, soft card
  background `#eff6ff`
- **Section nav rail**: sticky left, IntersectionObserver highlight, ≤ 200px wide
- **Mobile < 768px**: collapse rail → floating TOC button; stack Q1/Q2/Q3 cards

## Image inlining rules (sync.sh responsibility)

- For each `![Figure N: ...](../images/{id}/figN.png)` in notes: read
  `images/{id}/figN.png`, base64-encode, replace src with `data:image/png;base64,...`
- Images > 1 MB: warn but still inline (sync.sh should print a size
  report per paper)
- Missing images (notes references a fig that doesn't exist on disk):
  hard fail the render, don't silently drop

## Nav unifier (post-sync, MANDATORY)

After sync.sh, run the nav unifier snippet to ensure every HTML page
(papers / knowledge / tools / graph / index) has the canonical
`Papers | Knowledge | Graph | Tools | GitHub` nav. The exact script lives
in the site repo's toolchain; skill-side reminder is: if you touched any
HTML template, run it.

## Completeness gate (covers both notes and HTML)

```bash
python3 ~/.cursor/paper-db/tools/check_paper_completeness.py {id} --strict
```

Blocks Final Output if:
- `papers/{id}.html` missing or < 5 KB
- any `### Figure N:` in notes has no matching image
- any local path leaked into HTML
- KaTeX delimiter violations detected
